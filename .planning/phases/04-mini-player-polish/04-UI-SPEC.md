| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Mini-player vertical padding, thumb margin-right |
| md | 16px | Mini-player horizontal padding, border-radius |
| lg | 24px | Content area padding |

Touch target constraints (not spacing tokens): heart-button and play-button must be at least 44x44px tap area (WCAG). Achieved via padding on the element, not a spacing scale value.

## Color

All values are existing CSS custom properties. Phase 4 introduces no new colors.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | var(--bg) #0f0f13 | App background behind mini-player |
| Secondary (30%) | var(--surface-2) #252530 | Mini-player background |
| Accent (10%) | var(--accent) #ff3366 | Hairline progress bar, play-button color, spinner color, filled heart |
| Text | var(--text) #ffffff | Track title, play/pause emoji |
| Text muted | var(--text-muted) #9aa0a6 | Artist name |

Accent reserved for: hairline progress bar, play-button icon color, buffering spinner border, filled heart emoji (red).

Heart states:
- Empty: `\ud83e\udd0d` (white heart emoji) — inherits var(--text)
- Filled: `\u2764\ufe0f` (red heart emoji) — native red color

## Component Inventory

### Modified: MiniPlayer template

Current DOM order: `thumb -> info -> play-btn -> progress`

New DOM order: `thumb -> info -> heart-btn -> play-btn -> progress`

```
#mini-player (fixed, flex, align-items: center)
  img#mp-thumb .............. 42x42, border-radius 10px
  .mp-info .................. flex: 1, min-width: 0
    #mp-title ............... 13px/600, ellipsis
    #mp-artist .............. 11px/400, ellipsis, text-muted
  #mp-heart-btn ............. NEW — 26px emoji, padding 6px, touch-target 44px
  #mp-play-btn .............. 26px emoji, padding 6px 10px, accent color
  #mp-progress .............. absolute bottom, 2px height, accent
```

### New element: Heart button (#mp-heart-btn)

| Property | Value |
|----------|-------|
| Element | `<div id="mp-heart-btn" data-action="like">` |
| Content empty | `\ud83e\udd0d` |
| Content filled | `\u2764\ufe0f` |
| Font size | 26px (matches play-button) |
| Padding | 8px (results in ~42px visual, meets 44px tap area) |
| Min tap target | 44x44px (use padding to achieve) |
| Cursor | pointer |
| Transition | transform 0.15s ease-out on tap (:active scale(1.2)) |

Responsive: hide heart-button when viewport width < 340px via `@media (max-width: 339px) { #mp-heart-btn { display: none; } }` (Decision D-07).

### New behavior: Buffering spinner

Replaces play/pause emoji inside `#mp-play-btn` during buffering (Decision D-04).

| Property | Value |
|----------|-------|
| Trigger show | `waiting` event on active audio element |
| Trigger hide | `canplay` / `playing` event on active audio element |
| Visual | CSS border-spinner, 20x20px |
| Spinner border | 2px solid transparent |
| Spinner border-top | 2px solid var(--accent) |
| Border-radius | 50% |
| Animation | `spin 0.6s linear infinite` |
| Transition in/out | Instant replacement (no fade) — spinner replaces emoji text, emoji returns on canplay |

CSS keyframes:
```css
@keyframes spin {
  to { transform: rotate(360deg); }
}
.mp-spinner {
  display: inline-block;
  width: 20px;
  height: 20px;
  border: 2px solid transparent;
  border-top-color: var(--accent);
  border-radius: 50%;
  animation: spin 0.6s linear infinite;
  vertical-align: middle;
}
```

When buffering: `#mp-play-btn` innerHTML becomes `<span class="mp-spinner"></span>`
When resolved: `#mp-play-btn` innerHTML returns to `\u23f8` or `\u25b6` based on isPlaying state.

## Interaction Contract

### Existing gestures (no changes)

| Gesture | Target | Action | Threshold |
|---------|--------|--------|-----------|
| Tap | #mp-play-btn | Toggle play/pause | — |
| Tap | Anywhere else | Open fullscreen player | — |
| Swipe up | #mini-player | Open fullscreen player | dy > 40px, dy > dx |
| Swipe left | #mini-player | Next track | dx > 50px, dx > dy |
| Swipe right | #mini-player | Previous track | dx > 50px, dx > dy |

### New interactions

| Gesture | Target | Action | Details |
|---------|--------|--------|---------|
| Tap | #mp-heart-btn | Toggle like | Optimistic: swap emoji immediately, POST /library/like in background. On error: revert emoji. stopPropagation to prevent opening fullscreen. |

### State management

| Store key | Owner | Purpose |
|-----------|-------|---------|
| currentTrack | PlaybackEngine | Drives mini-player visibility and track info |
| isPlaying | PlaybackEngine | Play/pause icon state |
| isLiked | MiniPlayer/FullscreenPlayer (shared) | Heart fill state — new key needed |

`isLiked` must be set on track change (check against cached library likes list or infer from API response). Both MiniPlayer and FullscreenPlayer subscribe to it.

Buffering state: listen directly to audio element events (`waiting`/`canplay`/`playing`) via PlaybackEngine, not through Store. Rationale: transient UI state, no other subscriber needs it.
